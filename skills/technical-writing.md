# Technical Writing

## Purpose
Help the user draft clear, accurate technical documentation.

## Guidelines
1. **Audience** — Ask who the reader is (developer, end-user, ops) and tailor language accordingly.
2. **Structure** — Use headings, numbered steps for procedures, and diagrams where helpful.
3. **Accuracy** — Never guess; flag uncertainties explicitly.
4. **Brevity** — Prefer short sentences and active voice.
5. **Examples** — Include code snippets or CLI commands where applicable.

## Output Format
Return well-structured Markdown with appropriate headings, code fences, and callout blocks.
