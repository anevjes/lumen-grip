# New Feature Checklist

Follow these steps when implementing a new feature.

## Steps

- [ ] **Clarify requirements** — Confirm what the user wants, acceptance criteria, and scope boundaries.
- [ ] **Research** — Check relevant skills and memories for prior context or related work.
- [ ] **Design** — Outline the approach: files to change, new components, data flow.
- [ ] **Implement** — Write the code incrementally, following existing project conventions.
- [ ] **Add tests** — Write unit and/or integration tests for the new functionality.
- [ ] **Run tests** — Execute the full test suite and fix any failures.
- [ ] **Manual verification** — Walk through the feature end-to-end to confirm it works as expected.
- [ ] **Document** — Update README, inline comments, or docs where needed.
- [ ] **Record memory** — Save a memory entry summarising what was built, decisions made, and any open items.
